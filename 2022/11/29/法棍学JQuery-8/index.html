JQuery里的AJAX

AJAX允许通过与后台的web服务器交换数据来“异步”更新网页（浏览器页签不转圈圈）。这意味着可以更新网页的某些部分，而无需重新加载整个网页（浏览器页签转圈圈）。

使用AJAX，可以：

  • 将数据发送到Web服务器（在后台）
  • 从Web服务器读取数据（页面加载后）
  • 更新网页而无需重新加载页面

AJAX可以发送和接收各种格式的信息，包裹JSON、XML、HTML和文本文件，利用XMLHttpRequest对象实现与服务器的通信。

不同的浏览器对Ajax的实现方式不同，意味着采用典型的js方式来说会先Ajax，那么就要准备好为各种浏览器编写不同的代码，以确保Ajax功能能够在不同的浏览器上运行。

JQuery中考虑到了不同浏览器之间的差异，简化了Ajax的过程，使得不同的浏览器之间可以无缝实现Ajax。

Ajax load()方法

load()方法从服务器加载数据，并将返回的HTML放入所选的元素汇总，此方法提供了一种从Web服务器异步加载数据的简单方法。

语法：

$(selector).load(URL, data, callback)

参数：

  • URL 指定请求的URL
  • data （可选）指定与请求一起发送到服务器的纯对象或者字符串
  • callback （可选）指定在load()方法完成后执行的回调函数

load()下的callback参数

可选的callback参数指定load()方法完成后需要执行的回调函数。
callback的参数：

  • response 包含请求中的结果数据
  • status 包含请求的状态（success、notmodified、error、timeout、parsererror）
  • xhr 包含XMLHttpRequest对象

AJAX的$.get()方法与$.post()方法

jQuery $ .get()和$ .post()方法用于通过HTTP GET和POST请求从服务器请求数据。

在GET方法中，浏览器会将参数键值对添加到URL的末尾，用户能够在浏览器地址栏看到参数的具体情况，因此get相对不关心安全性。

  • get请求可以被缓存
  • get请求保留在浏览器的历史记录中
  • get请求可以添加书签
  • 处理敏感数据时，绝不使用get请求
  • get请求具有长度限制（2048个字符）

POST方法中，参数不会显示在浏览器的地址栏中，相对安全，因此如果表单数据包含敏感信息或者个人信息，应该始终使用POST

  • POST请求不会被缓存
  • POST请求不会保留在浏览器的历史记录中
  • POST请求无法添加书签
  • 处理敏感数据的时候应该使用POST请求
  • POST请求对数据长度没有限制

$.get()方法

语法：

$.get(URL,data,callback)

参数：

  • URL 指定请求的URL
  • data （可选）指定与请求一起发送到服务器的纯对象或者字符串
  • callback （可选）指定请求成功时执行的回调函数

$.post()方法

语法：

$.post(URL,data,callback)

参数：

  • URL 指定请求的URL
  • data （可选）指定与请求一起发送到服务器的纯对象或者字符串
  • callback （可选）指定请求成功时执行的回调函数

AJAX事件

Ajax请求会产生许多事件，按照类型分为：本地事件和全局事件

本地事件

本地时间可以在Ajax请求对象中订阅的回调

有四个Ajax本地事件：

  • beforeSend 在发送请求之前运行的函数
  • error 如果请求失败，则运行的函数
  • success 请求成功时要运行的功能
  • complete 完成请求后运行的功能（成功或失败都会触发）

全局事件

在文档上触发，调用任何正在侦听的处理程序

有六个Ajax全局事件：

  • ajaxComplete() -在Ajax请求完成时附加要调用的函数

  • ajaxError() -当Ajax请求完成并附有错误时，附加要调用的函数

  • ajaxSend() -在发送Ajax请求之前附加要调用的函数

  • ajaxStart() -附加第一个Ajax请求开始时要调用的函数

  • ajaxStop() -完成所有Ajax请求后，附加一个要调用的函数

  • ajaxSuccess() -每当Ajax请求成功完成时，附加要调用的函数
