什么是DOM遍历

DOM遍历基本上意味着在组成网页的DOM结点上查找。遍历DOM时，可以将文件树上移到任何所需的元素。还可以将层次结构下移到给定的某一个子节点，或者移动到同级结点。
下面给出的是HTML页面的树（DOM树）。使用JQuery遍历，可以轻松地在向上向下侧向移动。

遍历方法

遍历祖先（向上找）

  • parent()
  • parents()
  • parentsUntil()
parent()方法

调用了parent()方法后，在当前DOM结点上上移一个级别。

语法：

$(selector).parent(filter) // filter（可选） 指定选择器表达式以匹配元素

parents()方法

parents()方法会返回所选元素的所有祖先元素。

语法：

$(selector).parents(filter) // filter（可选）指定选择器表达式以匹配元素

parentUntil()方法

返回两个给定参数之间的所有祖先元素。

语法：

$(selector).parentsUntil(stop, selectorExpression)
参数：

  • stop（可选）
  • selectorExpression（可选）选择器表达式，元素或jQuery对象，以使元素与之匹配 注意：要返回多个祖先，需要用逗号分割每个表达式

遍历后代（向下找）

  • children()
  • find()
children()方法

返回所选的元素的直接子级。

语法：

$(selector).children(selectorExpression)

参数：

  • selectorExpression （可选）指定选择器表达式以匹配元素

selectorExpression可以寻找某一类特定的子代，例如：

find()方法

返回指定参数匹配的所有后代元素。

语法：

$(selector).find(selectorExpression)

参数：

  • selectorExpression （可选）选择器表达式、元素或者jQuery对昂，以将元素与之匹配 注意：要搜索多个后代，请用都还分割每个表达式

过滤

在通过遍历方法获取某一些DOM结点的时候，也可以通过过滤方法筛选出各种元素。

  • first()
  • last()
  • eq()
  • filter()
  • not()
  • has()
  • is()
first()方法

返回所选元素（能够选择到一个或多个）的第一个元素（选第一个）

last()方法

返回所选元素的最后一个元素

eq()方法

返回具有选定元素的特定索引号的元素

索引号始终从0开始，因此第一个数字将具有索引0，而不是1

filter()方法

返回与特定条件匹配的元素

此方法过滤所有不符合筛选条件的元素，并返回的那些匹配项

not()方法

返回不符合特定条件的元素

与filter()方法的作用相反

has()方法

返回与指定选择器匹配的所有元素，其中包含一个或者多个元素

is()方法

检查所选元素之一是否与给定参数匹配(选择的元素is给定参数?true:false)

如果这些元素中的至少一个与给定参数匹配，则此方法将返回true，否则返回false

过渡的类名

在进入、离开的过渡中，会有6个class切换

  1. v-enter-from：定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入后的下一帧移除；
  2. v-enter-from：定义进入过渡生效时的状态。在整个进入过渡的阶段中使用，在元素被插入之前生效，在过渡、动画完成之后移除。这个类可以用来定义进入过渡的过程时间、延迟和曲线函数；
  3. v-enter-to：定义进入过渡的结束状态。在元素被插入之后的下一帧生效（与此同时v-enter-from被移除），在过渡、动画完成后移除；
  4. v-leave-from：定义离开过渡的开始状态。在离开过渡被触发时立刻生效，下一帧被移除。
  5. v-leave-active：定义离开过渡生效时的状态。在整个离开过渡的阶段中应用，在离开过渡被触发时立刻生效，在过渡/动画完成之后移除。这个类可以被用来定义离开过渡的过程时间，延迟和曲线函数。
  6. v-leave-to：离开过渡的结束状态。在离开过渡被触发之后下一帧生效 (与此同时 v-leave-from 被移除)，在过渡/动画完成之后移除。

keep-alive

keep-alive是内置组件，不需要引入就可以直接使用，能够将重复渲染的组件缓存，提高性能。

keep-alive的开启会引起生命周期的变化：

  • 初次进入时：onMounted>onActivated
  • 退出后触发：deactivated
  • 再次进入：onActivated
  • 事件挂载的方法等，只执行一次的放在onMounted中；组件每次进去执行的方法放在onActivated中

keep-alive可以缓存组件自身的同时，也可以缓存组件上用户填写的一些信息，可以提高用户体验。

源码

位置：packages>runtime-core>src>components>KeepAlive.ts>KeepAlivelmpl

setup初始化，return一个渲染函数，读取了插槽的默认值，判断了length，说明keep-alive中只能有一个子节点，把子节点进行返回。

缓存机制：Map和Set进行key的缓存，cacheSubtree中进行缓存，通过pendingCacheKey在render后（因为在render后，所以第一次有onMounted执行）进行记录，存放到set中实现缓存，之后就会在set中找，不会再走onMounted。

再通过对缓存组件的一些标记判断，比如shapeFlag静态标记，以及keys.delete()、keys.add()进行更新缓存组件的活跃度。

对于生命周期，activate和deactivate是定义在sharedContext实例上的，经过patch更新一些属性后，将会在一个异步队列中执行这两个生命周期函数。
